Frequently Asked Questions

What output formats are available?

"Array of Objects" (most common) produces a JSON array where each row is an object with header names as keys. "Array of Arrays" skips headers and wraps each row as a simple array. "Keyed by first column" creates an object where each key is the value from the first column — useful for lookup tables.

What does "auto-cast numbers" do?

When enabled, values that look like numbers (integers or decimals) are converted from strings to actual JSON numbers. This avoids issues where "42" needs to be compared arithmetically. Disable it if your data contains IDs or zip codes that should remain as strings.

How are quoted fields and commas inside values handled?

The parser follows the RFC 4180 CSV standard. Fields wrapped in double-quotes can contain commas, newlines, and escaped double-quotes (""). The parser handles all of these cases correctly.

Is there a size limit?

There is no enforced limit. The converter runs in your browser, so the only constraints are your device's available memory and JavaScript engine. For very large files (100MB+) consider splitting them first.
