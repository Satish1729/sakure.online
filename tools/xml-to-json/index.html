Frequently Asked Questions

How are XML attributes handled?

Attributes are included as prefixed keys in the JSON object (default prefix "@"). For example, <book id="1"> becomes {"@id":"1",...}. You can change the prefix in options or disable attribute inclusion entirely.

What happens with repeated XML elements?

Repeated sibling elements with the same tag name are automatically converted to a JSON array. For example, multiple <item> elements become an array of item objects. Enable "Compact" to always use arrays, even for single elements.

How is mixed content (text + child elements) handled?

When an element contains both text and child elements, the text is stored under the configurable text key (default "#text") alongside the child element keys. Pure text-only elements are stored as a simple string value.

What about CDATA sections?

CDATA sections (<![CDATA[...]]>) are extracted and treated as text content. The raw CDATA wrapper is stripped and the inner text is included in the JSON output.

Is my XML data sent to a server?

No. The browser's built-in DOMParser handles all XML parsing client-side. Nothing is transmitted to any server, making this safe for internal or sensitive XML documents.
