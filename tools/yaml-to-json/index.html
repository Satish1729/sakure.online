Frequently Asked Questions

What YAML features are supported?

The converter supports scalars (strings, integers, floats, booleans, nulls), block mappings (objects), block sequences (arrays), inline flow style, multi-line strings (literal | and folded > blocks), quoted strings, and comments (stripped from output).

What about YAML anchors and aliases (& and *)?

Basic anchor/alias expansion is supported. Anchors (&name) define a value, and aliases (*name) reference it. This allows reusing shared values without duplication. Complex merge keys (<<:) are also handled.

How are YAML types mapped to JSON?

YAML true/false/yes/no → JSON true/false. YAML null/~/~ → JSON null. Integers and floats are parsed as numbers. Everything else becomes a JSON string. Use "Nulls as strings" to output null values as "null" strings if needed.

Can I convert multi-document YAML files?

Multi-document YAML (separated by ---) outputs the first document. If you need all documents, split them manually and convert each one separately.

Is my YAML data sent anywhere?

No. All processing happens client-side in your browser using JavaScript. Your YAML data never leaves your computer, making this tool safe for configuration files with secrets.
