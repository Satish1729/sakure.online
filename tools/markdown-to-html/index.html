Frequently Asked Questions

What is GitHub Flavored Markdown (GFM)?

GFM is an extension of standard Markdown that adds support for tables, task lists (checkboxes), strikethrough text, fenced code blocks with language tags, and autolinked URLs. It's the format used by GitHub READMEs, GitLab, and many documentation tools. This converter supports all GFM features.

Can I use this for blog posts or documentation?

Yes. Write your content in Markdown in the left pane, then copy the HTML from the right pane. The "Wrap in HTML" option adds a complete HTML document skeleton with basic CSS, making it suitable to paste directly into a blog CMS or save as a standalone file.

Is code syntax highlighting included in the output?

The live preview shows styled code blocks. The raw HTML output includes language class attributes (e.g. class="language-js") that are compatible with Prism.js and highlight.js — just include one of those libraries in your page to get full syntax highlighting.

What happens to unsafe HTML inside Markdown?

This tool renders HTML embedded in Markdown as-is in the preview. If you are generating HTML to be shown to other users on a website, always sanitize the output server-side or use a library like DOMPurify to prevent XSS attacks.
